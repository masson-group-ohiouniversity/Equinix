Overview

Equilibrist is a browser-based scientific application built with Python and Streamlit. It runs locally on your machine. You define a chemical system via a plain-text script, simulate titration or time-resolved curves, and optionally fit binding or rate constants to experimental data.

Key capabilities

  • Arbitrary equilibrium networks — any number of species, reactions, stoichiometries, and multiple products per reaction.
  • Kinetics simulation — irreversible (>) and reversible (<>) kinetic steps, including pre-equilibria defined with = in the same script.
  • Free-energy input mode — reactions can be defined using ΔG° (van 't Hoff) and ΔG‡ (Eyring) energy terms in kcal/mol instead of K or k values, with a user-specified temperature.
  • Liquid and solid titrant modes — volume-based additions with dilution, or solid additions with fixed cell volume.
  • UV-Vis global spectral fitting — Beer–Lambert analysis over any wavelength range; molar absorptivities solved analytically at each step.
  • NMR fitting — fast-exchange chemical shift, slow-exchange integration, and simultaneous mixed-mode fitting. Available in both equilibrium and kinetics modes.
  • Concentration fitting — direct fitting of measured species concentrations.
  • Dual optimizer (concentration mode) — L-BFGS-B (fast, gradient-based) followed by Nelder-Mead (robust, gradient-free), user-selectable per fit run.
  • User-defined soft bounds on K — restrict the optimizer to physically meaningful equilibrium constant ranges.
  • Free energy table — after fitting, ΔG° (equilibrium) and ΔG‡ (rate constants, Eyring) are reported in kcal/mol at the script temperature.
  • Publication-quality figure export — one-click PDF with vector text, Arial font, for subsequent editing in Adobe Illustrator.
  • Thermodynamic cycle detection — automatic warning when declared equilibrium constants are mutually inconsistent.
  • One-click data export — timestamped Excel file with concentrations and variables, plus PNG snapshot.
  • FAIR session save / restore — self-contained JSON bundle (script + data + fit state + uncertainty results + RNG seed + library versions) for reproducible archiving alongside a publication.
  • Information criteria (AIC, AICc, BIC) and Akaike weights — emitted automatically with every fit; F-test for nested model comparison.
  • Residual diagnostics — Durbin–Watson serial-correlation test and Shapiro–Wilk normality test, plus a four-panel figure (residuals vs predictor, histogram, normal Q–Q, metrics).
  • Identifiability tools — 1D RMSE profile (pin one, refit rest), 2D RMSE colormap (pin two, refit rest) with Hessian-derived confidence ellipses overlaid, parameter correlation heatmap, eigendecomposition of the Hessian covariance into stiff/sloppy directions, and per-parameter t-tests with user-editable null hypotheses.
  • Three independent uncertainty layers — non-parametric residual bootstrap (also parametric and Mammen wild), leave-one-out jackknife, and Monte Carlo propagation of user-declared input uncertainties (stock concentrations, titrant stocks, V₀). Each available for every fit mode; bootstrap and MC can run in parallel across all cores.
  • Local sensitivity (Masson ξ) — a 3N paired-difference grid over any user-selected subset of fitted and held-fixed quantities; returns conditional and marginal SEs and a coupling ratio.
  • Spectral rank analysis — Evolving Factor Analysis (forward + backward), Malinowski IND scree, and Target Factor Analysis on the absorbance matrix for any UV-Vis fit, as an independent cross-check on the model rank.
  • Shift-constraint syntax — equality groups (shift: A = B = C), monotone chains (shift: A > B > C), and per-column magnitude bounds in $constraints to tame Variable-Projection slack in NMR shift fits.
  • Acid-base titration mode — $reactions acid-base section accepts pKa values directly; Kw, pH variable, and water equilibrium are injected automatically.
  • Outlier flagging — click any experimental data point to exclude it from fitting; hollow markers distinguish excluded from included points; excluded points are absent from publication PDFs.
  • In-app script editor — edit the script directly in the browser without leaving the app.

Installation

Equilibrist requires Python 3.10 or newer. Two installation methods are available: conda (recommended, especially on Windows) and pip.

Method A — conda (recommended)

Best if you already have Anaconda or Miniconda. If not, download Miniconda (free, ~50 MB) from docs.conda.io/en/latest/miniconda.html.

🪟

Windows

  1. Open Anaconda Prompt (installed with Miniconda).
  2. Navigate to the Equilibrist folder: cd C:\path\to\Equilibrist
  3. Create the environment (once): conda env create -f environment.yml
  4. Activate it: conda activate equilibrist
🍎

macOS / Linux

  1. Open Terminal.
  2. Navigate to the Equilibrist folder: cd ~/path/to/Equilibrist
  3. Create the environment (once): conda env create -f environment.yml
  4. Activate it: conda activate equilibrist
Why conda?

The environment.yml pins Python to 3.11 and installs all dependencies in an isolated environment — it won't conflict with any existing Python on your machine. Binary packages (scipy, numpy, matplotlib) install more reliably via conda on Windows.

Open a terminal in the Equilibrist folder. If using conda, activate the environment first.

Streamlit starts a local server and opens your browser automatically at http://localhost:8501. The application runs entirely on your machine — no internet connection required after installation.

Every time you use Equilibrist

Script Structure

An Equilibrist script is a plain-text file (any extension, typically .txt) that fully defines the chemical system. It is divided into named sections, each opened by a $ keyword on its own line. Section names are case-insensitive. The order of sections does not matter to the parser, but is preserved in the exported files.

Lines beginning with # are comments. Inline comments after # anywhere on a line are stripped before parsing. Comment lines are preserved verbatim in the exported files. The keyword $equilibria is accepted as a synonym for $reactions.

A minimal script:

$concentrations
G0 = 1.00 mM

$volumes
V0 = 500 uL

$titrant
Ht = 10.00 mM

$reactions
G + H = GH; log K1 = 4.0

$plot
xmax = 3.00
x = H0/G0
y = G, H, GH

Upload this file with the Equilibrist Script uploader in the left sidebar, or type it directly into the in-app script editor. The simulation runs immediately and re-runs on every sidebar change. Uploading a new script performs a full session reset automatically.

$titrant

Defines what is added to the cell. Two modes are available, selected on the header line. Not used in kinetics mode.

Solution mode (default)

Each line names a component and its stock concentration. Adding solution incrementally increases the cell volume, diluting all species.

$titrant
Ht = 10.00 mM   # primary titrant, 10 mM stock
Lt = 5.00  mM   # co-dissolved ligand (optional)

Solid mode

Use $titrant solid. No stock concentration is needed. Cell volume is fixed at V₀ throughout. An optional ratio after a semicolon sets stoichiometric fractions for mixed solid additions.

$titrant solid
Ht          # single solid component
$titrant solid
Ht; 1.0     # two-component mixture, mole ratio H : M = 1.0 : 1.3
Mt; 1.3

$temperature

Sets the temperature used for free-energy calculations. This affects two things: (1) conversion of ΔG values to K or k when using the free-energy input mode in $reactions, and (2) the ΔG° / ΔG‡ values shown in the Free energies table after fitting. The default is 298.15 K (25 °C) when the section is absent.

Three equivalent syntaxes are accepted:

$temperature
T = 25 C        # 25 °C  (recommended syntax)
$temperature
T = 310.15 K    # 310.15 K
$temperature 37 C   # inline — value and unit on the header line
UnitMeaning
KKelvin (default when no unit given)
CCelsius (converted to Kelvin internally)
Temperature and fitting

The temperature does not affect equilibrium or kinetics calculations — K and k values stored in the script are always the direct constants. Temperature only controls (a) the DG→K/k conversion at parse time, and (b) the ΔG display in the fit statistics panel.

$reactions

Defines all chemical reactions — equilibria and/or kinetics — in a single section. The arrow symbol determines the reaction type and the solver engaged.

ArrowTypeSolver
=Thermodynamic equilibriumEquilibrium solver
>Irreversible kinetic stepODE (scipy.integrate)
<>Reversible kinetic stepODE (scipy.integrate)
$equilibria alias

$equilibria is accepted as a full synonym for $reactions. Scripts using the old keyword remain valid.

Equilibrium reactions ( = )

reactants = products; log Kname = value

Parameter names (K1, K2, …) must be unique across the entire script. The value is log₁₀(K). You may also provide the linear K directly — Equilibrist converts automatically:

$reactions
G + H  = GH;  log K1 = 4.5       # log form
GH + H = GH2; K2 = 31623          # linear form, auto-converted
2 G + H  = G2H; log K3 = 5.1       # 2:1 stoichiometry

Stoichiometric coefficients are integers separated from the species name by a space: 2 G, 3 H. The space is mandatory — 2G without a space is interpreted as a species literally named 2G. Multiple products are allowed: G + H = GH_A + GH_B.

Soft bounds on equilibrium constants

You can restrict the optimizer to a physically meaningful range for each K by appending a from … to … clause. Bounds outside this range are penalised quadratically — the optimizer is strongly discouraged from leaving the region, but the constraint is not hard. The bound values follow the same convention as the K itself: log units if you wrote log K1, linear units if you wrote K1.

SyntaxMeaning
from lo to hiBoth lower and upper soft bounds
from loLower bound only
to hiUpper bound only
(absent)Default bounds: log K ∈ [−2, 14]
 
$reactions
L + Eu = EuL;  log K1 = 6.0;  from 2.0 to 12.0
L + Eu = EuL;  K1 = 1e6;     from 1e2 to 3.5e12  # linear bounds

Kinetic reactions ( > and <> )

As soon as any kinetic arrow appears in the script, Equilibrist switches to the ODE engine for the whole system. Rate constants can be entered in log form (log k1 = 3.0) or linear form (k1 = 1000). Both forms are accepted.

$reactions
A + B >  AB; log k1  = 3.0                  # irreversible, log form
A + B >  AB; k1  = 1000                   # irreversible, linear form
A + B <> AB; log k1  = 4.0; log k-1 = 1.0  # reversible
A + B <> AB; k1  = 2e4;  k-1 = 2.0         # reversible, linear

Mixed equilibria and kinetics

Both arrow types can coexist freely. Pre-equilibria (=) in a mixed system are enforced via large reversible rate constants consistent with the declared K value.

$reactions
G + H = GH;  log K1 = 4.0    # fast pre-equilibrium
GH    > P;   log k2 = -2.0   # slow irreversible step

Free-energy input mode (DG / DG#)

Instead of K or k values, reactions can be defined using free energies in kcal/mol. The temperature set in $temperature (default 298.15 K) is used for all conversions. Two prefixes are available:

PrefixMeaningUsed for
DG = valueReaction free energy ΔG° (kcal/mol), van 't Hoff: K = exp(−ΔG°/RT)Equilibrium reactions (=)
DG# = valueActivation free energy ΔG‡ (kcal/mol), Eyring: k = (k_BT/h) · exp(−ΔG‡/RT)Kinetic reactions (>, <>)

For reversible kinetic reactions (<>), both DG# (forward barrier) and DG (reaction energy) are required. The reverse barrier is computed as ΔG‡_rev = ΔG‡ − ΔG°. An error is raised if this would be ≤ 0 (unphysical: transition state below products).

DG and k/K inputs cannot be mixed on the same reaction line.

$temperature
T = 25 C

$reactions
# Equilibrium via ΔG° (van 't Hoff)
G + H = GH; DG = -5.5              # ΔG° = −5.5 kcal/mol → log K ≈ 4.04

# Irreversible kinetics via ΔG‡ (Eyring)
GH > P; DG# = 18.0                  # ΔG‡ = 18.0 kcal/mol

# Reversible kinetics via ΔG‡ and ΔG° (Eyring + van 't Hoff)
A + B <> C; DG# = 15.0; DG = -3.0  # ΔG‡_rev = 15.0 − (−3.0) = 18.0 kcal/mol
Parameter names in DG mode

When DG mode is used, parameter names are auto-generated: equilibrium constants are named K + label (e.g. K1), forward rate constants kf + label, reverse rate constants kr + label. An optional label can be appended to the DG prefix: DG#A generates kfA.

Parameter naming (general)

Parameter names can be any alphanumeric string — log K1, log Benzene, log beta3, k-1 are all equally valid. The distinction between an equilibrium constant and a rate constant is determined entirely by the arrow symbol (= vs > / <>), not by the name. Names must be unique across the entire script.

Acid-base mode ($reactions acid-base)

A dedicated section header for acid-base titration systems. Instead of writing out proton-transfer equilibria by hand, each line specifies an acid/base pair and its pKa. Equilibrist automatically builds the deprotonation reactions, injects the water autoionization equilibrium (Kw), and defines a pH variable.

Three line formats

Auto-injected elements (invisible to the user)

  • Water equilibrium — H2O = H + OH; Kw with logK = −14.0, added once.
  • Water concentration — H2O0 = 1 M understood as a unitary activity, hidden from the sidebar.
  • pH variable — pH = -log(H * 1e-3), available in $plot y and $variables. (Concentrations are in mM internally; the factor 1e-3 converts to M before taking the log.)

Sidebar behaviour in acid-base mode

  • The parameter section is headed pKa values instead of "Equilibrium constants".
  • Each pKa is displayed and entered as a positive number (e.g. 4.76); the solver receives logK = −pKa.
  • Kw is hidden — it is implicit and not user-editable.
  • H₂O is hidden from the concentrations list.

Solid acid-base mode

Use $reactions acid-base and combine with $titrant solid. Cell volume is fixed at V₀ throughout.

$nmr

Activates NMR fitting mode. One or both of the following keywords can appear in the section. Available in both equilibrium (titration x-axis) and kinetics (time x-axis) modes — the data format differs only in column A (see Kinetics Data Files).

shift: — Fast exchange (chemical shift titration)

The observed shift is a population-weighted average over all species. The target must be a species name or a $variables name whose expression gives the stoichiometric composition of the observed nucleus.

$nmr
shift: Gtot         # Gtot = G + GH + GH2 defined in $variables
shift: Gtot, Htot   # multiple independent signal targets

Data column headers must match the target name exactly, or use Name.N / Name_N suffixes for duplicate proton environments (e.g. Gtot.1, Gtot_2). The first data row sets δ_free for each column. Bound-state shift increments Δδ are solved analytically (linear least squares) inside each optimizer step; only log K (or log k) is optimised non-linearly.

integration: — Slow exchange (signal integrals)

Separate peaks for each species are integrated. Absolute concentrations are recovered using the host mass balance. The integers after the colon give the number of NMR-active nuclei contributing to each signal column, in order.

$nmr
integration: 3, 3, 6   # columns B, C, D: 3H, 3H, 6H respectively

Column headers should match species names (e.g. G, GH) optionally with .N or _N suffixes for duplicate peaks. The primary component (mass-balance denominator) is the first species listed in $concentrations.

Mixed mode (simultaneous slow + fast exchange)

Include both keywords in the same section. The first n data columns (where n = count of numbers in integration:) are integration columns; all remaining columns are chemical shift columns.

$nmr
integration: 3, 3   # columns B–C: slow-exchange integrals
shift: Gtot          # columns D onward: fast-exchange shifts

read: — Pin pure-species shifts from a second sheet

When the data workbook carries known intrinsic chemical shifts for one or more species on a second sheet, list those species after read: in $nmr. Equilibrist reads their per-column shifts verbatim from sheet 2 and pins the corresponding intrinsic-shift rows during the Variable Projection step; the remaining species' shifts are still solved analytically at each fit iteration.

$nmr
shift: Gtot, Htot
read: GH, GH2    # intrinsic δ of GH and GH2 read from sheet 2

Sheet 2 layout mirrors the UV-Vis "known spectra" tab: row 1 contains the signal-column headers, column A lists species names, body cells carry intrinsic ppm values. Cells left empty (or with the literal string NaN) mark that particular species × column combination as still free to fit, so a partial pin is supported.

noref — Release the auto-pinned reference

By default, Equilibrist treats the first species in each fast-exchange target group as the math reference and auto-pins its intrinsic shift increment to dd[ref] = 0. The Variable-Projection linear sub-problem is then solved over the non-reference species only, and the reported "pure-species chemical shifts" are absolute ppm: δpure = δobs(at first x) + dd. This is the legacy behavior, identical to v1.0, and is what most users want.

Add a single bare line noref to $nmr to release that auto-pin. Every species' intrinsic shift then enters the linear sub-problem on equal footing — including the reference. What happens next depends on whether a read: anchor is present in the same section:

ConfigurationShift scalePure-species shift table reports…
Default (no noref)Anchored by auto-pinning the reference species to dd = 0.Absolute ppm: δobs(V=0) + dd. Reference row matches the V=0 observed shift.
noref + read: anchorAnchored externally by the pinned species in sheet 2 (pure_shifts_anchored = True).Absolute ppm. The reference species' intrinsic shift is now fitted from the data, not assumed equal to its V=0 observed value.
noref alone, no read:Undetermined — the linear system carries a uniform-shift symmetry (any constant added to every dd leaves δobs unchanged).Δδ relative to the math reference, with the table header explicitly relabelled "Δδ relative to math reference". The absolute scale cannot be recovered.
 
$nmr
shift: Gtot
read: GH2      # pure-species δ of GH2 read from sheet 2 → anchor
noref            # release auto-pin; GH2 now anchors the absolute scale
When to use noref

Reach for noref when (a) you have an independently measured pure-species shift for at least one species — typically a saturated complex from a sample with a large excess of titrant — that you want to enter via read: as the absolute anchor, and (b) you do not want to assume the first species' intrinsic shift equals its V=0 observed value. For straightforward titrations this assumption is harmless and the default mode is preferred.

noref without read: is a diagnostic, not a publication output

Running noref alone (no read:) still converges — the equilibrium constants and the relative pure-species shift differences are identifiable — but the absolute pure-species shifts are not. The output table is relabelled "Δδ relative to math reference" to make this clear. The numeric fitted constants are unchanged by the choice of anchor.

$spectra

Activates UV-Vis/circular dichroism/emission global spectral fitting using the Beer–Lambert law. Available in both equilibrium and kinetics modes. Three optional keywords may appear inside the section:

$spectra
transparent: H, Eu       # these species do not absorb
path = 0.2              # optical path length in cm (default 1.0)
read: cage               # read the spectrum of cage from sheet 2 of the data file
$spectra                 # valid: section with no body — all species may absorb

transparent / non-emissive

Species listed after transparent: are excluded from the absorber list and their molar absorptivity is fixed at ε = 0 at every wavelength. The synonym non-emissive: is also accepted (useful for fluorescence or CD experiments).

path = value

Sets the optical path length in cm. This value is used to convert the analytically solved absorptivity matrix from Beer–Lambert units (mM⁻¹, path absorbed) to true molar absorptivities ε (mM⁻¹ cm⁻¹) for display on the pure-species spectra plot and for export. It does not affect which constants are optimised; it is a display and export scaling factor only. Default: 1.0 cm.

read: — providing known spectra

Species listed after read: have their molar absorptivity ε(λ) provided by the user in a second sheet of the uploaded data file, rather than solved by the optimizer. Providing a known spectrum does not remove a species from the fitting problem — Equilibrist still uses Beer–Lambert to determine K. It simply constrains the ε of that species to the provided values, leaving fewer free parameters for the spectral decomposition.

$spectra
transparent: H
read: cage                # cage spectrum read from sheet 2
path = 1.0
$spectra
read: A, B, C             # multiple known spectra

Rules for read::

  • Species listed in read: but absent from sheet 2 of the data file are silently ignored — no error is raised. Only species that are both listed in read: and found in sheet 2 have their spectrum pinned.
  • Species listed in transparent: and in read: simultaneously remain transparent (ε = 0 everywhere). The transparent: designation takes priority.
  • Empty cells in sheet 2 (NaN values) are treated as "no data at this wavelength": ε is not pinned there and is solved freely instead. Empty cells are never interpreted as ε = 0. This means a partial spectrum (e.g., measured over 300–400 nm only, with the remaining wavelengths blank) pins ε only in the measured region and leaves the rest unconstrained.
  • Wavelengths in sheet 2 that fall outside the wavelength range of the experimental data are not used.

Shift constraints — NMR fast-exchange only

Variable Projection (VP) in NMR shift mode solves a linear sub-problem X · d ≈ y at every iteration of the non-linear optimizer, where d is the vector of intrinsic species shifts (relative to the reference species). When many species feed into a single observed shift column, that linear sub-problem has slack — it can absorb speciation errors that should otherwise constrain log K's. The shift: constraint lines below restrict d so the slack disappears and the data has to pay for itself in log K rather than in pure-species shifts.

Fitting

All modes minimise the sum of squared residuals (SSR) over the selected log₁₀(K) or log₁₀(k) parameters. Outlier-flagged points are excluded from the dataset before the fit begins — they contribute no residuals.

Re-fit loop (all NMR and kinetics modes)

The optimizer runs multiple Nelder-Mead passes in a loop. After each pass, parameter bounds shift to ±20% of the current fitted value ("bounds recentring"), and the simplex step size is set to the full feasible range per dimension. The loop continues until parameters stop moving (max change < tolerance) or the global timeout is reached. In integration and mixed modes, the back-calculated concentrations (bc) are frozen at the start of each pass to avoid a moving-target problem — bc is only recomputed between passes.

Concentration mode

The equilibrium is solved directly at each experimental x-point. Both L-BFGS-B and Nelder-Mead are available. When both are on, L-BFGS-B runs first; Nelder-Mead then refines from the best point found. In addition to log K values, initial concentrations and titrant stock concentrations can be fitted simultaneously by ticking their fit checkboxes in the sidebar.

NMR chemical shift mode

Mole fractions F_i(x) are fully determined for fixed K. Bound-state increments Δδ_i are solved analytically inside each step — only K is optimised non-linearly. Uses the re-fit loop with bounds recentring.

NMR integration mode

Back-calculated concentrations are derived from integrals using mass balances (K-independent). The optimizer moves K until theoretical curves match bc. Uses the re-fit loop with frozen bc per pass and bounds recentring.

NMR mixed mode

Integration and shift residuals combined into a single weighted objective. Integration bc is frozen per pass; shift SSR is recomputed each iteration. Uses the re-fit loop.

UV-Vis spectral mode

For fixed K, the concentration matrix C is fully determined. ε is solved analytically at each optimizer step: for species with no known spectrum ε is found by lstsq or NNLS; for species declared with read: whose spectrum was loaded from sheet 2, the corresponding ε rows are pinned to the provided values wavelength by wavelength (NaN entries in the known spectrum are treated as free). L-BFGS-B runs first, followed by Nelder-Mead. The optimizer selection checkboxes do not affect spectral fitting — both stages always run.

Kinetics modes

ODE integration (scipy) at each evaluation. All NMR and spectra fitting modes are available in kinetics mode, using time [s] as the x-axis instead of equivalents:

  • Concentration — directly fit measured species concentrations vs. time.
  • NMR shift (kinetics, fast exchange) — population-weighted average δ vs. time.
  • NMR integration (kinetics, slow exchange) — absolute concentrations from peak areas vs. time.
  • NMR mixed (kinetics, simultaneous slow + fast) — v2.0. The same dataset may contain both integration and shift columns; Equilibrist weights the two residual streams into a single objective and reports R² / RMSE per stream. Triggered automatically when $nmr contains both integration: and shift: lines in a kinetic script.
  • UV-Vis / fluorescence spectra — time-resolved absorbance matrices with Variable-Projection-solved ε.

The same re-fit loop applies to every kinetics mode.

Kinetics fitting speed

Each objective evaluation requires a full ODE integration (~20 ms on a typical machine). Fitting 3 or more parameters (e.g. k₁, G₀, H₀ simultaneously) may use the full timeout budget. This is expected — the optimizer tracks the best point seen and returns it when time expires, so a timed-out fit is still a valid fit. Increase the Timeout value in the sidebar if you want more optimizer time.

Soft bounds — applied as quadratic penalties outside [lo, hi]. Default range: log K ∈ [−2, 14] for equilibrium constants; no default bounds for rate constants.

Decision flow — which uncertainty to report

The asymptotic Hessian SE printed alongside each fitted constant assumes (i) i.i.d. normal residuals, (ii) the optimum is interior to all bounds, and (iii) the SSR surface is locally parabolic. If Durbin–Watson sits cleanly near 2, Shapiro–Wilk passes (p ≥ 0.05), and no constraint or bound is binding, the Hessian SE is trustworthy. When Shapiro fails, use the residual bootstrap. When DW < 1.5, use the wild bootstrap. When the data is fine but the experimental inputs (stocks, V₀) carry meaningful prep-error, run a Monte Carlo alongside the bootstrap — input uncertainty often dominates for supramolecular titrations and is invisible to the residual-noise methods.

Information criteria — AIC, AICc, BIC

Information criteria let you compare two or more candidate models fitted to the same data on a single scalar that balances goodness-of-fit against model complexity. Equilibrist computes all three after every successful fit and writes them under the fit summary as a single line:

• AIC = 142.18   |   AICc = 142.34   |   BIC = 148.91

Formulas

Under independent Gaussian residuals with variance σ² estimated by σ̂² = SSR/n, the maximised log-likelihood (dropping additive constants that cancel in any pairwise comparison) is

ℓmax = −(n / 2) · ln(SSR / n)

From which:

CriterionFormulaUse when
AIC [Akaike 1974]AIC = n · ln(SSR / n) + 2 pn / p ≥ 40
AICc [Hurvich–Tsai 1989]AICc = AIC + 2 p (p + 1) / (n − p − 1)Small-sample correction; always safe to use
BIC [Schwarz 1978]BIC = n · ln(SSR / n) + p · ln(n)When stronger penalty on extra parameters is desired

Here n = number of data points entering the fit, p = number of free fitted parameters. The residual variance σ² is treated as a nuisance parameter and is not counted in p.

Lower is better — but only differences matter

Absolute AIC/BIC values are dominated by the additive constants we dropped, so a single AIC of 142 means nothing on its own. Compare two fits to the same data: ΔAIC = AICworse − AICbetter ≥ 0. A rule of thumb (Burnham & Anderson 2002): ΔAIC ≤ 2 → models are essentially equivalent; 4 ≤ ΔAIC ≤ 7 → considerably less support for the higher model; ΔAIC > 10 → essentially no support for the higher model.

Durbin–Watson and Shapiro–Wilk

The same line of the fit summary that prints AIC/AICc/BIC carries the Durbin–Watson statistic d and the Shapiro–Wilk p-value:

• Durbin–Watson = 1.79 (✓); Shapiro p = 0.038 (non-normal)

Durbin–Watson tests neighbour-to-neighbour autocorrelation in the ordered residual sequence:

d = Σi=2..n (ri − ri−1)² / Σ ri²
RangeIndication
1.5 ≤ d ≤ 2.5✓ Consistent with the independent-residuals assumption.
d < 1.5⚠ Positive autocorrelation. Smooth trend left in residuals — often an under-parameterised model, a baseline drift, or a missing equilibrium / kinetic step.
d > 2.5⚠ Negative autocorrelation. Rare in titration data; can indicate over-smoothing or weight mis-specification.

Shapiro–Wilk tests residual normality: W is the squared correlation between the ordered residuals and the standard-normal quantiles they would be predicted to follow. W ∈ [0, 1] with W near 1 indicating normality. The associated p-value rejects the null of normality when p < 0.05.

Why this matters for reporting

The Hessian-based standard errors printed in the Fitted constants table assume Gaussian residuals. When Shapiro fails, the bootstrap (next section) is the recommended fallback — it captures the actual sampling distribution without that assumption.

Parameter correlation heatmap

The Hessian covariance is a local-quadratic approximation. For nonlinear or constraint-bounded problems the 2D profile is the trustworthy diagnostic; the heatmap is a fast screen.

Bootstrap confidence intervals

Toggle 🔁 Show bootstrap confidence intervals. The Hessian SEs printed alongside each parameter assume Gaussian residuals and a locally-parabolic SSR surface. Bootstrap drops both assumptions: it estimates the sampling distribution of every fitted parameter empirically by repeating the fit on many synthetic datasets generated from the original one, then reads percentile CIs off that empirical distribution.

Non-parametric residual bootstrap (default)

At each iteration, residuals from the converged fit {r1, …, rn} are resampled with replacement and added to the model-predicted values ŷi to form synthetic yi*. The full fit is repeated on every synthetic dataset, yielding one set of fitted parameters per iteration. This is the procedure used by Bindfit (Hibbert & Thordarson 2016) and is the recommended default.

Parametric Monte Carlo

Synthetic data are generated by adding Gaussian noise to the model-predicted values, with σ estimated from the residuals: yi* = ŷi + N(0, σ̂²). Faster and lower-variance than the residual bootstrap, but only valid when the Shapiro–Wilk test passes — read the normality flag first.

Wild bootstrap (Mammen 1993)

For heteroscedastic data, residuals are scaled by a random multiplier u drawn from Mammen's two-point auxiliary distribution:

u ∈ { −(√5 − 1)/2, +(√5 + 1)/2 } with probabilities { (√5 + 1)/(2√5), 1 − (√5 + 1)/(2√5) }

This preserves the original residual's magnitude (it is scaled, not resampled) but randomly flips its sign, which is robust to variance that depends on the predictor. Use this when the residual vs predictor panel shows a fan shape.

When to prefer jackknife over bootstrap

Jackknife is the right tool when (i) you suspect a single observation is a leverage point and want to see its effect quantified, or (ii) you have a small dataset (n < 30) where the percentile bootstrap CI may be unstable. Bootstrap is otherwise more accurate (jackknife systematically underestimates SE for non-smooth statistics).

Monte Carlo — experimental input uncertainty

Toggle 🎲 Show Monte Carlo (experimental-uncertainty propagation). Local sensitivity test (Masson ξ)

Toggle 🎚️ Show Local sensitivity test (Masson). The local sensitivity test (LST) is an Equilibrist-original diagnostic that probes how the RMSE responds when an arbitrary user-selected subset of N variables — fitted parameters and/or held-fixed quantities — is jointly perturbed in log space. Where bootstrap and MC isolate one source of uncertainty at a time, the LST quantifies the coupling between any pair of variables.

Definition

The user selects N variables (any subset of fitted K's, fitted k's, fitted concentrations, fitted titrants, held-fixed concentrations, held-fixed titrants, V₀) and a step δ in log10 units. Equilibrist evaluates the RMSE on a 3N grid: each variable takes the three values {−δ, 0, +δ} around its optimum, all combinations swept. For every selected variable X, the paired-difference average

ξX = ⟨ ( RMSE(Xopt + δX, others) − RMSEopt ) / RMSEopt ⟩

is computed over the 2·3N−1 paired differences in which X is perturbed by ±δ while the other N − 1 variables run through all 3N−1 configurations of their levels (cf. manuscript eq. 20).

Spectral rank analysis — EFA, scree, TFA

Toggle 📊 Show data-rank analysis (EFA / scree / TFA) after a UV-Vis spectra fit. This panel is unique to spectra fits and operates directly on the absorbance matrix D (rows = titration / time steps, columns = wavelengths), independently of the chemical model. It answers the question: how many distinct absorbing species does the data support, and do my fitted pure spectra ε agree with the data's intrinsic factor space?

Three sub-tools are reported on a single three-panel figure.

Evolving Factor Analysis (EFA)

For each row index k = 1, …, nobs, EFA computes the SVD of the leading k rows (forward direction) or the trailing k rows (backward direction) of D, and records the resulting singular values σi(k). Plotting σi vs k shows how each chemical species' contribution rises above the noise floor as titration steps are added (forward) or removed (backward). The combined forward + backward view defines the concentration window of each species — the titration range over which it carries detectable absorbance.

Malinowski IND scree

The full-matrix SVD yields one decreasing sequence of singular values σ1 ≥ σ2 ≥ … For each candidate rank k Malinowski defines the residual standard deviation

RE(k) = √( Σj>k σj² / (r·(c−k)) )

with r = max(n_obs, n_ch), c = min(n_obs, n_ch), and the indicator

IND(k) = RE(k) / (c − k)²

The recommended rank is the k that minimises IND. The scree panel plots σk (log scale) and the IND curve on the same axis; in practice the cleanest signature of the true rank is a sharp drop in IND that flattens to a plateau.

The panel also reports the estimated noise floor (geometric mean of the trailing-half singular values) and a signal-to-noise ratio σk / σnoise for each k.

Target Factor Analysis (TFA)

For a candidate rank r (default: the IND-argmin), the data column-space projector is Pcol = Vr VrT, where Vr is the first r right singular vectors of D. Any putative pure spectrum t can be tested by projecting it onto Pcol and reading off the relative residual ||t − Pcolt|| / ||t||. A small residual confirms the target is reproducible from the data's leading singular vectors — irrespective of which chemical model produced it.

Equilibrist auto-feeds the fitted ε spectra (from the most recent UV-Vis fit) as TFA targets, so the rank-analysis figure simultaneously shows (i) the data's intrinsic rank, (ii) the noise floor, and (iii) which fitted pure species are consistent with that rank.

Cross-checks

The IND-argmin rank should match (or be at most one above) the number of independently absorbing species declared in $reactions (with transparent: species removed). When IND-argmin < model rank, the data does not support the declared model — typically because two species' spectra are too similar and the κ(C) warning will already be firing. When IND-argmin > model rank, the data carries an extra component the chemical model is missing (an impurity, a hidden equilibrium, or baseline drift).

Restoring a session

Drop the .json file onto the same uploader that accepts .txt scripts (the script-upload control automatically detects JSON). Equilibrist verifies the SHA-256 checksum of the embedded data, restores the script and data into the editor and uploader badges, reinstates every fit_state control to its saved value, and rehydrates each uq.* block into its diagnostic panel. The bootstrap CI table, the jackknife influence plot, and the Monte Carlo histograms re-render without needing to be re-run.

Publication archival pattern

Save a session immediately before submitting a manuscript. The single JSON file, deposited in a repository alongside the paper, lets any reviewer or reader reconstruct every reported number without needing to re-run anything, on the same Equilibrist version and (via the env block) the same NumPy / SciPy stack. The script is human-readable, the data is verifiably the same bytes you analysed, and every uncertainty layer comes back with the exact CI you cited.

Backward compatibility

v1 session files saved with earlier development builds are still loadable, but they lack the env, seed, and uq blocks. Reloading a v1 file will give you the script, data, and fit state, but the uncertainty panels will read as empty until you re-run them.

Toggle 🎚️ Show Local sensitivity test, select {log K₁, log K₂}, step δ = 0.2 log. Equilibrist evaluates 3² = 9 grid points. The test returns:

  • σcond(log K₁) ≈ 0.18, σcond(log K₂) ≈ 0.20 — the SE each constant would have if the other were known exactly. Comparable in size to the bootstrap σ.
  • Coupling σmarg/σcond ≈ 2.3 for both — the marginal Hessian SE is roughly 2 × inflated by the (still independent but loose) joint identification.

Chemical conclusion

What the numbers actually say about the chemistry

Even at the tighter bootstrap intervals (σ ≈ 0.27 – 0.28), the two binding constants overlap substantially. The fitted Δlog K = 0.08 is well within sampling noise and should not be over-interpreted as a chemically meaningful inequivalence between the two binding events. The "correct" reportable statement is: "log K₁ = 4.60, log K₂ = 4.68 (bootstrap 95 % CI 4.04–5.09 and 4.25–5.34 respectively); the two binding events are not statistically distinguishable on this dataset."

Step 2 — 2D RMSE profiles for the flagged pairs

Run the 2D profile twice, once on (k₁, k₋₁) and once on (k₁, k₂):

  • (k₁, k₋₁): a long valley extends predominantly along the k₋₁ axis with a modest diagonal tilt. The pre-equilibrium constant K is well constrained; the individual rate constants are coupled but distinguishable.
  • (k₁, k₂): a roughly round well with modest tilt. The two are coupled but each is independently identified by the data.

Step 3 — Sloppy-spectrum eigenanalysis

Toggle 🔬 Show parameter identifiability (sloppy spectrum). Equilibrist returns three eigendirections with σᵢ ranging from 3 × 10⁻⁴ for v₁ (stiffest) to 2 × 10⁻³ for v₃ (sloppiest).   • v₁ (stiffest, σ = 3 × 10⁻⁴) is the best-determined linear combination: coefficients (−0.77, +0.23, +0.59) on (k₁, k₋₁, k₂). The data fixes this particular combination very tightly.
  • v₃ (sloppiest, σ = 2 × 10⁻³) is overwhelmingly aligned with k₋₁ alone (weight −0.97). k₋₁ is the parameter whose value is most weakly constrained.
  • The σalone bar chart ranks k₋₁ as the parameter that would benefit most from external information (e.g. from an independent equilibrium measurement of K).
