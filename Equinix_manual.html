Overview

Equinix is a browser-based scientific application built with Python and Streamlit. It runs locally. You define a chemical system via a plain-text script, simulate titration or time-resolved curves, and optionally fit binding or rate constants to experimental data.

Key capabilities

  • Arbitrary equilibrium networks — any number of species, reactions, stoichiometries, and multiple products per reaction.
  • Kinetics simulation — irreversible (>) and reversible (<>) kinetic steps, including pre-equilibria.
  • Liquid and solid titrant modes — volume-based additions with dilution, or solid additions with fixed cell volume.
  • UV-Vis global spectral fitting — Beer–Lambert analysis over any wavelength range; molar absorptivities solved analytically at each step.
  • NMR fitting — fast-exchange chemical shift, slow-exchange integration, and simultaneous mixed-mode fitting.
  • Concentration fitting — direct fitting of measured species concentrations.
  • Dual optimiser — L-BFGS-B (fast, gradient-based) followed by Nelder-Mead (robust, gradient-free), user-selectable per fit run.
  • User-defined soft bounds on K — restrict the optimiser to physically meaningful equilibrium constant ranges.
  • Publication-quality figure export — one-click PDF with vector text, Arial font, for subsequent editing.
  • Thermodynamic cycle detection — automatic warning when declared equilibrium constants are mutually inconsistent.
  • One-click data export — timestamped Excel file with concentrations and variables, as well as PNG snapshot.

Installation

Equinix requires Python 3.10 or newer. Two installation methods are available: conda (recommended, especially on Windows) and pip.

Method A — conda (recommended)

Best if you already have Anaconda or Miniconda. If not, download Miniconda (free, ~50 MB) from docs.conda.io/en/latest/miniconda.html.

🪟

Windows

  1. Open Anaconda Prompt (installed with Miniconda).
  2. Navigate to the Equinix folder: cd C:\path\to\Equinix
  3. Create the environment (once): conda env create -f environment.yml
  4. Activate it: conda activate equinix
🍎

macOS / Linux

  1. Open Terminal.
  2. Navigate to the Equinix folder: cd ~/path/to/Equinix
  3. Create the environment (once): conda env create -f environment.yml
  4. Activate it: conda activate equinix
Why conda?

The environment.yml pins Python to 3.11 and installs all dependencies in an isolated environment — it won't conflict with any existing Python on your machine. Binary packages (scipy, numpy) install more reliably via conda on Windows.

Running Equinix

Open a terminal in the Equinix folder. If using conda, activate the environment first.

conda activate equinix     # conda users only — skip if using pip
streamlit run app.py

Streamlit starts a local server and opens your browser automatically at http://localhost:8501. The application runs entirely on your machine — no internet connection required after installation.

Every time you use Equinix

conda users: conda activate equinix then streamlit run app.py.
pip users: just streamlit run app.py.

Windows shortcut

Right-click the Desktop → New → Shortcut. Enter: cmd /k "conda activate equinix && streamlit run C:\full\path\to\app.py". Double-clicking launches Equinix in one step.

Press Ctrl + C in the terminal to stop the server.

Script Structure

An Equinix script is a plain-text file (any extension, typically .txt) that fully defines the chemical system. It is divided into named sections, each opened by a $ keyword on its own line. Section names are case-insensitive. The order of sections does not matter to the parser, but is preserved in the exported files.

Lines beginning with # are comments. Inline comments after # anywhere on a line are stripped before parsing. Comment lines are preserved verbatim in the exported files. The keyword $equilibria is accepted as a synonym for $reactions.

A minimal script:

$concentrations
G0 = 1.00 mM

$volumes
V0 = 500 uL

$titrant
Ht = 10.00 mM

$reactions
G + H = GH; log K1 = 4.0

$plot
xmax = 3.00
x = Ht/G0
y = G, H, GH

Upload this file with the Equinix Script uploader in the left sidebar. The simulation runs immediately and re-runs on every sidebar change. Uploading a new script (without clicking Reset) performs a full session reset automatically.

$concentrations

Defines the initial concentrations of all species present in the cell/NMR tube before titration begins.

Name = value unit

Species name derivation

If the name ends in 0, the trailing zero is stripped to give the free-species symbol used by the solver. G0 → species G.

UnitMeaningNote
MMolar
mMMillimolarDefault when no unit given
uMMicromolar

All concentrations are stored and displayed in mM internally. Scientific notation is accepted (e.g. 1e-3 M).

$concentrations
G0   = 1.00 mM    # free-species name: G
Dye0 = 10 uM      # free-species name: Dye
Use in x-axis expression

The script name (e.g. G0) is available in the x-axis expression as the initial number of moles (mmol = C₀ × V₀). Dividing a cumulative titrant mmol by this gives a dimensionless equivalents axis: x = Ht/G0.

$volumes

Sets the initial cell/NMR tube volume. Only V0 is required.

$volumes
V0 = 500 uL
UnitMeaningNote
LLitres
mLMillilitresDefault when no unit given
uLMicrolitres

In solution mode, V₀ is the starting volume; dilution is recalculated at every titration point. In solid mode, V₀ is fixed throughout.

$titrant

Defines what is added to the cell. Two modes are available, selected on the header line.

Solution mode (default)

Each line names a component and its stock concentration. Adding solution incrementally increases the cell volume, diluting all species.

$titrant
Ht = 10.00 mM   # primary titrant, 10 mM stock
Lt = 5.00  mM   # co-dissolved ligand (optional)

Solid mode

Use $titrant solid. No stock concentration is needed. Cell volume is fixed at V₀ throughout. An optional ratio after a semicolon sets stoichiometric fractions for mixed solid additions.

$titrant solid
Ht          # single solid component
$titrant solid
Ht; 1.0     # two-component mixture, mole ratio H : M = 1.0 : 1.3
Mt; 1.3

Rule (applied in order)ExampleFree species
Ends in t → strip itHtH

$reactions

Defines all chemical reactions — equilibria and/or kinetics — in a single section. The arrow symbol determines the reaction type and the solver engaged.

ArrowTypeSolver
=Thermodynamic equilibriumEquilibrium solver
>Irreversible kinetic stepODE (scipy.integrate)
<>Reversible kinetic stepODE (scipy.integrate)
$equilibria alias

$equilibria is accepted as a full synonym for $reactions. Scripts using the old keyword remain valid.

Equilibrium reactions ( = )

reactants = products; log Kname = value

Parameter names (K1, K2, …) must be unique across the entire script. The value is log₁₀(K). You may also provide the linear K directly — Equinix converts automatically:

$reactions
G + H  = GH;  log K1 = 4.5       # log form
GH + H = GH2; K2 = 31623          # linear form, auto-converted
2 G + H  = G2H; log K3 = 5.1       # 2:1 stoichiometry — note the space after 2

Stoichiometric coefficients are integers separated from the species name by a space: 2 G, 3 H. The space is mandatory — 2G without a space is interpreted as a species literally named 2G, not as 2 molecules of G. This allows species names containing numbers (such as GH2M or CO2) to be unambiguous. Multiple products are allowed: G + H = GH_A + GH_B.

Soft bounds on equilibrium constants

You can restrict the optimiser to a physically meaningful range for each K by appending a from … to … clause after the K definition (separated by a semicolon). Bounds outside this range are penalised quadratically — the optimiser is strongly discouraged from leaving the region, but the constraint is not hard.

The bound values follow the same convention as the K itself: if you wrote log K1 = 6.0, bounds are in log units; if you wrote K1 = 1e6, bounds are in linear units.

SyntaxMeaning
from lo to hiBoth lower and upper soft bounds
from loLower bound only
to hiUpper bound only
(absent)Default bounds: log K ∈ [−2, 14]
 
$reactions
# log scale bounds
L + Eu = EuL;  log K1 = 6.0;  from -2.0 to 14.0

# linear scale bounds (auto-converted to log)
L + Eu = EuL;  K1 = 1e6;     from 1e2 to 3.5e12

# upper bound only
L + Eu = EuL;  log K1 = 6.0;  to 14.0

# lower bound only
L + Eu = EuL;  K1 = 3e5;     from 1e4
Why bounds matter for UV-Vis

Without bounds the optimiser can escape to unphysical values (e.g. log K = 26) where the Beer-Lambert lstsq reconstruction is still near-perfect but the K values are meaningless. Soft bounds keep the fit physically grounded. The default range [−2, 14] is suitable for most solution-phase equilibria.

Kinetic reactions ( > and <> )

Rate constant names can be any alphanumeric string. As soon as any kinetic arrow appears in the script, Equinix switches to the ODE engine for the whole system.

$reactions
A + B >  AB; log k1  = 3.0                  # irreversible, M⁻¹s⁻¹
A + B <> AB; log k1  = 3.0; log kr1 = 1.0  # reversible

Mixed equilibria and kinetics

Both arrow types can coexist freely. Pre-equilibria in a mixed system are enforced via large reversible rate constants consistent with the declared K value.

$reactions
G + H = GH;  log K1 = 4.0    # fast pre-equilibrium
GH    > P;   log k2 = -2.0   # slow irreversible step
Parameter naming

Parameter names can be any alphanumeric string — log K1, log Benzene, log beta3 are all equally valid. The distinction between an equilibrium constant and a rate constant is determined entirely by the arrow symbol (= vs > / <>), not by the name. Names must be unique across the entire script.

$variables

Defines algebraic combinations of species concentrations for use as plot quantities, NMR shift targets, or in the x-axis expression. Full arithmetic is supported: +, -, *, /, parentheses.

$variables
Gtot  = G + GH + GH2              # total host in all forms
alpha = (GH + GH2) / (G + GH + GH2)  # fractional saturation

Variables are computed from solved concentrations, not fitting parameters. They can appear in the y = list of $plot and as targets in shift: of $nmr. Variables may reference other variables; circular dependencies raise a parse error.

$plot

Controls the x-axis expression, sweep range, and y-axis species list.

$plot
xmax = 3.00
x    = Ht/G0
y    = G, H, GH, GH2
KeyDescription
xmaxMaximum x-axis value. Controls both the simulation sweep range and the plot axis limit. Automatically updated when a data file is uploaded (set to the x-value of the last data point, rounded up). In kinetics mode this sets the time axis maximum in seconds. The sidebar input overrides the script value.
xx-axis expression. In kinetics mode the x-axis is always time in seconds and this key is not used. In equilibrium mode any arithmetic expression of species names, concentration names, and titrant names is valid (see namespace table below).
yComma-separated list of species or $variables to plot. All species are plotted if omitted.

$nmr

Activates NMR fitting mode. One or both of the following keywords can appear in the section.

shift: — Fast exchange (chemical shift titration)

The observed shift is a population-weighted average over all species. The target must be a species name or a $variables name whose expression gives the stoichiometric composition of the observed nucleus.

$nmr
shift: Gtot         # Gtot = G + GH + GH2 defined in $variables
shift: Gtot, Htot   # multiple independent signal targets

Data column headers must match the target name exactly, or use Name.N / Name_N suffixes for duplicate proton environments (e.g. Gtot.1, Gtot_2). The first data row sets δ_free for each column. Bound-state shift increments Δδ are solved analytically (linear least squares) inside each Nelder-Mead step; only log K is optimised non-linearly.

integration: — Slow exchange (signal integrals)

Separate peaks for each species are integrated. Absolute concentrations are recovered using the host mass balance. The integers after the colon give the number of NMR-active nuclei contributing to each signal column, in order.

$nmr
integration: 3, 3, 6   # columns B, C, D: 3H, 3H, 6H respectively

Column headers should match species names (e.g. G, GH) optionally with .N or _N suffixes for duplicate peaks. The primary component (mass-balance denominator) is the first species listed in $concentrations.

Mixed mode (simultaneous slow + fast exchange)

Include both keywords in the same section. The first n data columns (where n = count of numbers in integration:) are integration columns; all remaining columns are chemical shift columns. Both residual types are combined into a single weighted objective function optimised in one Nelder-Mead run.

$nmr
integration: 3, 3   # columns B–C: slow-exchange integrals
shift: Gtot          # columns D onward: fast-exchange shifts

$spectra

Activates UV-Vis/circular dichroism/emission global spectral fitting using the Beer–Lambert law. Optionally declare which species are transparent (ε = 0 at all wavelengths):

$spectra
transparent: H, Eu       # these species do not absorb
$spectra
transparent:             # valid: empty list — all species may absorb
$spectra                 # also valid: section with no body

All species not listed as transparent are treated as absorbers with unknown, wavelength-dependent molar absorptivities ε(λ). The optical path length is absorbed into ε — no separate path-length input is required.

When $spectra is present and a data file is uploaded, Equinix:

  • Displays the rainbow spectral plot below the main chart (violet = first spectrum → red = last).
  • Updates xmax to the x-value of the last spectrum.
  • Populates the wavelength range boxes from the file's actual wavelength extent.

Main Plot

The main chart shows simulated concentration curves [mM] vs. the evaluated x expression, clipped to [0, xmax]. After fitting, back-calculated data points are overlaid as filled circles in matching colors:

  • Spectra mode — points labelled Species (UV-Vis), derived from back-projection A = C·ε.
  • NMR integration — points labelled Species (NMR), from mass-balance deconvolution.
  • Concentration mode — experimental mM values overlaid directly.

UV-Vis rainbow spectral plot

Below the main chart when UV-Vis data are loaded. X-axis: wavelength [nm], clipped to the sidebar range. Y-axis: absorbance. Each spectrum is colored on a violet→red gradient (violet = first addition, red = last). The first and last spectra are labelled with their x-axis value.

Pure species spectra plot

After UV-Vis fitting, a third plot shows the fitted molar absorptivity ε(λ) [mM⁻¹] for each absorbing species. This is the Beer–Lambert ε matrix solved analytically at the best-fit K values.

NMR chemical shift plot

After NMR shift fitting, a plot shows the experimental Δδ points and fitted Δδ curves for each signal column. Y-axis: Δδ (ppm).

Fitting

All modes minimise the sum of squared residuals (SSR) over the selected log₁₀(K) or log₁₀(k) parameters. The optimiser strategy depends on the data type.

Concentration mode

The equilibrium is solved directly at each experimental x-point. Theoretical species concentrations [mM] are compared to measured values. Both L-BFGS-B and Nelder-Mead are available (user-selectable via checkboxes). When both are on, L-BFGS-B runs first; Nelder-Mead runs subsequently only if L-BFGS-B did not improve the objective.

NMR chemical shift mode

Mole fractions F_i(x) are fully determined for fixed K. Bound-state increments Δδ_i are solved analytically by linear least squares inside each step — only K is non-linearly optimised. Uses Nelder-Mead only.

NMR integration mode

Absolute concentrations are back-calculated from integrals using mass balances (K-independent, performed once). The optimiser then moves K until theoretical curves match the back-calculated concentrations. Uses Nelder-Mead only.

NMR mixed mode

Integration and shift residuals are combined into a single weighted objective. Uses Nelder-Mead only.

UV-Vis spectral mode

For fixed K, the concentration matrix C (n_spectra × n_absorbers) is fully determined. The molar absorptivity matrix ε (n_absorbers × n_wavelengths) is solved analytically: ε = lstsq(C, A). Only K values are optimised non-linearly. L-BFGS-B runs first as a fast warm-up, followed by Nelder-Mead for robustness. This two-stage strategy is always used regardless of the checkbox settings (UV-Vis fitting ignores the checkboxes).

Soft bounds are applied as a quadratic penalty outside the user-defined [lo, hi] range for each parameter. The default range is log K ∈ [−2, 14] for all parameters without explicit bounds.

UV-Vis timeout and identifiability

UV-Vis fitting runs a background timer. If the Timeout (s) limit is reached, fitting stops and the best parameter set found so far is returned — the K values update and a yellow warning banner is shown. The timeout applies to all fit modes.

After every UV-Vis fit, Equinix analyses whether the fitted K values are uniquely identifiable from the spectral data by computing the condition number of the concentration matrix C:

  • cond(C) = σ_max / σ_min, where σ are the singular values of C.
  • If cond(C) > 10,000, the concentration profiles of the complexes are nearly proportional — the individual K values cannot be separated from the spectral data alone. A warning is displayed with the condition number and the best-determined parameter combination (typically the cumulative product log(K₁·K₂·…·K_n)).

Auto-optimize wavelength range

When ticked, a two-pass procedure selects the most informative wavelength sub-range:

1

Full-range fit (pass 1)

Fits over the complete [min, max] range to obtain an initial ε(λ) estimate for all absorbers.

2

Range selection

Spectral contrast = variance of ε across all absorbers at each wavelength. Edge wavelengths below the 10th percentile of contrast are trimmed. A 50 nm minimum window is enforced.

3

Refined fit (pass 2)

Re-fits on the selected range. Wavelength boxes update automatically.

Fit Statistics

Displayed in the right column after each fit. A green banner confirms success; a yellow banner appears if fitting timed out or did not fully converge.

StatisticDescription
Spectral R²(spectra only) Coefficient of determination for the absorbance reconstruction A_calc = C·ε vs. A_measured. Almost always close to 1.0 regardless of K quality — the Beer-Lambert lstsq solution minimises absorbance residuals exactly. It does not measure whether K values are physically correct.
Concentration R²(spectra only) Coefficient of determination between the equilibrium-model concentrations C_model and the concentrations derived from the absorbance data C_spectral (back-projected through ε). This is the more meaningful quality metric — a poor fit on the main plot corresponds to a low Concentration R².
RMSERoot mean squared error in the observable's units: mM (concentration/integration), ppm (NMR shift), AU (UV-Vis absorbance).
Data pointsTotal number of data values used (spectra × wavelengths for UV-Vis; total measurements for NMR/concentration).
Reduced χ²SSR / (n_data − n_params). Shown only when n_data > n_params.
Interpreting UV-Vis R²

Spectral R² ≈ 1.0 is normal and expected even for a completely wrong K — the lstsq solver always finds an ε that reconstructs the absorbances well. Use Concentration R² and the visual quality of the main plot to judge the fit quality.

The Fitted log K table shows each parameter with its log₁₀(K), linear K, and ±error (from finite-difference Hessian). Values are written back to the sidebar inputs immediately.

If the condition number diagnostic detects high correlation, a warning is shown with the condition number and the best-determined combination of K values (e.g. log(K₁·K₂·K₃)).

After integration or mixed-mode fitting, a Back-calculated concentrations table lists the recovered species, the number of raw signals used, and the n_H values.

After shift or mixed-mode fitting, a Pure-species chemical shifts (ppm) table reports δ for every species per signal column.

The PDF axis limits are determined server-side from the Plotly layout's declared range (i.e. xmax and the corresponding y extent). Thermodynamic Cycle Detection

On every render, Equinix traces all reaction paths between every pair of species. If two independent paths give different total log₁₀(K) (tolerance 0.01 log units), a warning banner names the inconsistent species. No warning means all K values are mutually consistent. Circular dependencies (a species appearing in its own reaction ancestry) are also detected.

Example: Sequential 1:3 Binding with UV-Vis Fitting

Eu³⁺ binds ligand L step-by-step. Eu does not absorb in the measured range; L and the three complexes do. Soft bounds prevent the optimiser from escaping to unphysical values.

$concentrations
Eu0 = 0.10 mM

$volumes
V0 = 2400 uL

$titrant
Lt = 2.00 mM

$reactions
L   + Eu  = EuL;  log K1 = 6.0; from 2.0 to 12.0
L   + EuL = EuL2; log K2 = 5.0; from 2.0 to 12.0
L   + EuL2= EuL3; log K3 = 4.0; from 2.0 to 12.0

$plot
xmax = 10.0
x    = Lt/Eu0
y    = Eu, L, EuL, EuL2, EuL3

$spectra
transparent: Eu
Identifiability note

When the absorption spectra of EuL, EuL2 and EuL3 are very similar, the individual K values cannot be determined from spectral data alone (the concentration matrix C is ill-conditioned). Equinix will report a correlation warning with the condition number. Only the cumulative product K₁·K₂·K₃ (= β₃) is well-determined in that case.
